## ❗ 常见问题

### 问题1：git pull 权限错误
**错误信息**: `insufficient permission for adding an object`

**解决方案**:
```bash
sudo git fetch origin master
sudo git reset --hard origin/master
```

### 问题2：容器没有运行
**症状**: `docker compose ps` 显示为空

**解决方案**:
```bash
docker compose up -d
```

### 问题3：构建失败
**解决方案**: 查看错误日志，修复代码后重新部署
```bash
docker compose logs --tail=100
```

### 问题4：网站没有更新
**可能原因**:
- 浏览器缓存
- 容器没有重新构建

**解决方案**:
1. 清除浏览器缓存（Ctrl + Shift + R）
2. 重新构建容器：
```bash
docker compose down
docker compose build --no-cache
docker compose up -d
```

## 🔍 故障排查详细步骤

### 如何判断部署是否成功

1. **检查容器状态**
```bash
docker compose ps
# 应该看到 kaoshanmeng-app 容器状态为 Up
```

2. **检查日志**
```bash
docker compose logs --tail=30
# 应该看到 "Ready in XXXms" 和 "Local: http://localhost:3000"
```

3. **测试本地访问**
```bash
curl -I http://localhost:3000
# 应该返回 200 OK
```

4. **访问网站**
打开浏览器访问 https://kaoshanmeng.cn，清除缓存后刷新（Ctrl + Shift + R）

### 常见错误代码含义

- **Exit code 1**: 构建失败，通常是代码语法错误或依赖问题
- **Exit code 137**: 内存不足，容器被 OOM Killer 杀死
- **Exit code 255**: SSH 连接失败或权限问题

### 构建失败排查

如果 `docker compose build` 失败：

1. **查看完整错误日志**
```bash
docker compose build 2>&1 | tail -100
```

2. **检查常见问题**
   - TypeScript 类型错误
   - 缺少依赖包
   - 导入路径错误
   - JSX 语法错误

3. **清除 Docker 缓存重试**
```bash
docker compose down
docker system prune -a  # 清除所有未使用的镜像和缓存
docker compose build --no-cache
docker compose up -d
```

### 容器启动后立即退出

1. **查看退出原因**
```bash
docker compose ps -a
docker compose logs
```

2. **可能原因**
   - 环境变量配置错误
   - 端口被占用
   - 数据库连接失败

3. **检查端口占用**
```bash
sudo lsof -i :3000
# 如果端口被占用，杀死进程或修改端口
```
