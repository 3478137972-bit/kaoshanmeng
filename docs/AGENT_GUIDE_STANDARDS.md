# 员工引导消息标准规范

## 概述
本文档定义了靠山盟系统中所有员工引导消息的格式标准，确保用户体验的一致性和可读性。

## 格式标准

### 1. 文本格式要求

#### ✅ 推荐做法
- 使用**数字编号**（1. 2. 3.）来组织信息结构
- 使用**纯文本**，避免使用Markdown语法（如 `**加粗**`）
- 适度使用**中文标点符号**【】来标记关键词
- 保持**简洁友好**的对话语气

#### ❌ 避免做法
- 不使用 `**文本**` Markdown加粗语法（会显示为星号）
- 不使用表情符号（可能在某些环境显示异常）
- 不使用复杂的Markdown格式（表格、代码块等）

### 2. 消息结构模板

```
盟主你好，我是[员工名称]。为了帮你[核心目标]，我需要收集以下信息：

1. [信息项标题]
[详细说明和示例]

2. [信息项标题]
[详细说明和示例]

3. [信息项标题]
[详细说明和示例]
```

### 3. CSS渲染要求

在渲染引导消息时，必须添加 `whitespace-pre-line` CSS类来保留换行符：

```tsx
<p className="text-sm leading-relaxed whitespace-pre-line">
  {message.content}
</p>
```

**原因**：HTML默认会将换行符折叠成空格，导致文本挤成一坨，影响阅读体验。

## 实际示例

### ✅ 正确示例

```typescript
"定位诊断师": `盟主你好，我是定位诊断师。为了帮你找到最适合的商业定位，我需要收集以下信息：

1. 职业经历与高光时刻
请描述主要职业经历、岗位职责及取得的关键成就（如：负责过百万级App）

2. 专业技能与熟练度
列出专业技能并评估等级（如：数据分析-精通、文案-中等）

3. 核心优势/擅长解决的问题
你最擅长在什么场景下解决什么麻烦？（如：将模糊需求拆解为执行方案）`
```

### ❌ 错误示例

```typescript
// 问题：使用了Markdown加粗和表情符号
"定位诊断师": `盟主你好，我是定位诊断师。为了帮你找到最适合的商业定位，我需要收集以下信息：

📋 **职业经历与高光时刻**
请描述主要职业经历、岗位职责及取得的关键成就（如：负责过百万级App）

🎯 **专业技能与熟练度**
列出专业技能并评估等级（如：数据分析-精通、文案-中等）`
```

## 配置位置

所有员工引导消息配置在：
```
components/dashboard/chat-console.tsx
```

在 `agentGuideMessages` 对象中添加或修改员工的引导消息。

## 添加新员工引导消息

1. 在 `agentGuideMessages` 对象中添加新的键值对
2. 键名必须与 `sidebar.tsx` 中的员工名称完全一致
3. 按照上述格式标准编写引导消息
4. 确保使用数字编号和清晰的分段
5. 提供具体的示例帮助用户理解

## 测试检查清单

添加或修改引导消息后，请检查：

- [ ] 文本中没有 `**` 星号显示
- [ ] 换行符正确显示，不是挤成一坨
- [ ] 数字编号清晰可见
- [ ] 每个信息项之间有适当的空行
- [ ] 示例内容具体且易于理解
- [ ] 语气友好、专业
